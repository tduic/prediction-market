================================================================================
CORE ARCHITECTURE HIGHLIGHTS
================================================================================

SIGNAL FLOW:
1. Core service generates trading signal
2. Signal pushed to Redis queue as JSON
3. Execution service consumes via BRPOP
4. SignalHandler validates schema & parameters
5. OrderRouter routes legs to platform clients
6. PolymarketClient/KalshiClient execute orders
7. Results logged to database
8. PositionStateManager tracks fills
9. PnL calculated and persisted

EXECUTION MODES:
- Simultaneous: asyncio.gather for parallel leg execution
- Sequential: execute leg A, wait for fill, then execute leg B

RETRY STRATEGY:
- Max 3 attempts per order
- Exponential backoff: 1s, 2s, 4s
- Logged at each attempt
- Full error context preserved

IDEMPOTENCY:
- Signal ID tracked in memory (Set)
- Query order_events table for database check
- Duplicate signals rejected with warning
- Prevents accidental double-execution

DATA PERSISTENCE:
- Write-through caching for position state
- Periodic flush to SQLite (configurable interval)
- ACID transactions for all database operations
- INSERT OR REPLACE for idempotent updates

ERROR HANDLING:
- Pydantic validation errors
- Network timeouts (httpx)
- Contract errors (web3.py)
- HTTP status errors (Kalshi)
- Graceful degradation
- Structured logging at all levels

MONITORING:
- Latency metrics (submission & fill times)
- Signal event tracking
- Order event logging
- Comprehensive debug logging
- Error count tracking

================================================================================
RUNNING THE SYSTEM
================================================================================

1. START REDIS:
   docker-compose up -d redis

2. INITIALIZE DATABASE:
   python -c "from core.storage.db import run_migrations; import asyncio; asyncio.run(run_migrations(...))"

3. START CORE SERVICE:
   REDIS_URL=redis://localhost:6379 python -m core.main

4. START EXECUTION SERVICE:
   REDIS_URL=redis://localhost:6379 python -m execution.main

5. LOAD PAIRS:
   python scripts/seed_pairs.py --file config/pairs_seed.json

6. BACKFILL PRICES:
   python scripts/backfill_prices.py --platform polymarket --since 30

7. VALIDATE PAIRS:
   python scripts/validate_pairs.py --status unverified

8. EXPORT ANALYTICS:
   python scripts/export_analytics.py --output analytics_export
